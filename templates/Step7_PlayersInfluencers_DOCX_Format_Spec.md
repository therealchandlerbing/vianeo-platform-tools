## Page 1 Structure

### 1. Document Title

**Content:** `[Project Name]: Ecosystem Analysis`

**Formatting:**
- Font: Calibri
- Size: 48pt
- Style: Bold
- Color: #1a1a1a (RGB: 26, 26, 26) - Near black
- Alignment: Left
- Space after: 6pt

**Character Limit:** Project name should be 100 characters maximum (total title length variable)

**Example:** `Rayla AI: Ecosystem Analysis`

---

### 2. Subtitle

**Content:** `Players & Influencers | Vianeo Framework`

**Formatting:**
- Font: Calibri
- Size: 24pt
- Style: Italic (NOT bold)
- Color: #5a5a5a (RGB: 90, 90, 90) - Medium gray
- Alignment: Left
- Space after: 12pt

**Fixed Text:** This subtitle is standardized across all documents

---

### 3. Executive Context Paragraph

**Content:** 2-3 sentences explaining scope and interpretation

**Formatting:**
- Font: Calibri
- Size: 20pt
- Style: Regular (not bold, not italic)
- Color: #3a3a3a (RGB: 58, 58, 58) - Dark gray
- Alignment: Left
- Space after: 18pt

**Character Limit:** 500 characters maximum (WITH spaces)

**Purpose:** Explain what market/ecosystem this covers, define Players vs. Influencers briefly, and indicate what acceptability ratings mean

**Example:**
```
This analysis maps critical stakeholders in the grant writing AI market. Players are active market participants who buy, sell, or provide services. Influencers shape market dynamics through policy, standards, or advocacy. Acceptability ratings indicate alignment with Rayla AI's innovation.
```

### 4. Ecosystem Players Section

#### Section Header

**Content:** `Ecosystem Players`

**Formatting:**
- Font: Calibri
- Size: 28pt
- Style: Bold
- Color: #2c3e50 (RGB: 44, 62, 80) - Dark blue-gray
- Alignment: Left
- Space before: 6pt
- Space after: 12pt

#### Players Table

**Structure:** 3-column table with 1 header row + 8-10 data rows

**Table Style:** Custom (not built-in Word style)

**Column Configuration:**

| Column | Header Text | Width (DXA) | Width (inches) | Width (approx %) |
|--------|-------------|-------------|----------------|------------------|

#### Table Header Row Formatting

**Background Color:** #34495e (RGB: 52, 73, 94) - Blue-gray

**Text:**
- Font: Calibri
- Size: 22pt
- Style: Bold
- Color: #ffffff (RGB: 255, 255, 255) - White
- Alignment: Left (all columns)

**Cell Padding:**
- All sides: 160 DXA (~0.11 inches)

**Header Cells:**
- Column 1: "Player"
- Column 2: "Acceptability"
- Column 3: "Strategic Implication"

#### Table Data Rows Formatting

**Row Count:** 8-10 rows (exactly)

**Cell Padding:**
- Top: 100 DXA
- Bottom: 100 DXA
- Left: 140 DXA
- Right: 140 DXA

**Borders:**
- Color: #ecf0f1 (RGB: 236, 240, 241) - Very light gray
- Style: Single line
- Width: 4pt (minor borders)

##### Column 1: Player Name

**Content:** Entity name (max 60 characters)

**Formatting:**
- Font: Calibri
- Size: 20pt
- Style: Bold
- Color: #2c3e50 (RGB: 44, 62, 80) - Dark blue-gray
- Alignment: Left
- Vertical alignment: Center

**Background:** White (no shading)

##### Column 2: Acceptability

**Content:** One of four values:
- Favorable
- Neutral
- Unfavorable
- Don't know

**Formatting:**
- Font: Calibri
- Size: 20pt
- Style: Bold
- Alignment: Center (horizontal)
- Vertical alignment: Center

**Color-Coded Background and Text:**

| Rating | Background Color | Text Color |
|--------|------------------|------------|
| **Favorable** | #d5f4e6 (RGB: 213, 244, 230) - Soft green | #27ae60 (RGB: 39, 174, 96) - Green, bold |
| **Neutral** | #fff4d9 (RGB: 255, 244, 217) - Soft yellow | #f39c12 (RGB: 243, 156, 18) - Amber, bold |
| **Unfavorable** | #fadbd8 (RGB: 250, 219, 216) - Soft red | #c0392b (RGB: 192, 57, 43) - Red, bold |
| **Don't know** | #f5f5f5 (RGB: 245, 245, 245) - Light gray | #7f8c8d (RGB: 127, 140, 141) - Medium gray, bold |

**Critical:** Both background AND text color must be applied for visual clarity

##### Column 3: Strategic Implication

**Content:** Strategic note (max 250 characters WITH spaces)

**Formatting:**
- Font: Calibri
- Size: 19pt
- Style: Regular (not bold, not italic)
- Color: #4a4a4a (RGB: 74, 74, 74) - Medium-dark gray
- Alignment: Left
- Vertical alignment: Top (or Center if space allows)
- Text wrapping: Enabled (multi-line if needed)

**Background:** White (no shading)

**Note Structure:** [Current Situation] + [Impact] + [Strategic Implication]

## Spacing and Layout Guidelines

### Vertical Spacing

| Element | Space Before | Space After |
|---------|--------------|-------------|
| Document Title | 0pt | 6pt |
| Subtitle | 0pt | 12pt |
| Executive Context | 0pt | 18pt |
| Section Headers | 6pt | 12pt |
| Tables | 0pt | 12pt |

### Cell Spacing

**Header Cells:**
- Uniform: 160 DXA all sides

**Data Cells:**
- Top/Bottom: 100 DXA
- Left/Right: 140 DXA

### White Space Principles

1. **Generous margins:** 0.75" creates breathing room
2. **Consistent spacing:** Predictable rhythm between elements
3. **Clear hierarchy:** Larger elements (title) more space, smaller elements less
4. **Scannable tables:** Cell padding ensures text doesn't feel cramped

## Visual Quality Standards

### Professional Presentation Checklist

- [ ] Clean, uncluttered layout
- [ ] Consistent color coding (acceptability ratings)
- [ ] Appropriate white space (not cramped)
- [ ] Professional typography (Calibri, proper hierarchy)
- [ ] Scannable tables (easy to read at a glance)
- [ ] Color accessibility (sufficient contrast ratios)
- [ ] Print-ready (looks good on paper, not just screen)
- [ ] Exactly 2 pages (not 1.5, not 3)

### BCG-Level Standards

**What BCG quality means:**
1. **Strategic clarity:** Every element serves a purpose
2. **Visual hierarchy:** Most important information stands out
3. **Professional restraint:** No unnecessary decoration
4. **Data density:** High information content without feeling dense
5. **Executive-ready:** Can be shared with C-suite or board without revision

### Common Quality Issues to Avoid

❌ **Too cramped:** Text touching borders, no breathing room
❌ **Inconsistent formatting:** Different fonts, misaligned elements
❌ **Color overload:** Too many colors, distracting
❌ **Generic appearance:** Looks like default Word document
❌ **Poor contrast:** Text hard to read against background
❌ **Overflow to page 3:** Didn't respect 2-page constraint
❌ **Weak notes:** Vague or generic strategic implications

### Method 2: Manual Creation in Word

**Steps:**
1. Create blank document
2. Set page size and margins (File → Page Setup)
3. Add title with formatting (48pt Calibri bold, #1a1a1a)
4. Add subtitle (24pt Calibri italic, #5a5a5a)
5. Add executive context (20pt Calibri regular, #3a3a3a)
6. Insert 3-column table (1 header + 8-10 data rows)
7. Format header row (22pt bold white on #34495e)
8. Set column widths (approximately 3.33", 2.5", 7.17")
9. Fill in player data
10. Apply color-coded backgrounds to Acceptability column
11. Format entity names (20pt bold #2c3e50)
12. Format strategic notes (19pt regular #4a4a4a)
13. Insert page break
14. Repeat steps 6-12 for Influencers section
15. Verify: exactly 2 pages, all formatting correct

**Challenges:**
- Time-consuming
- Easy to make formatting errors
- Manual validation of character limits
- Requires design eye for BCG quality

### Method 3: Word Template

**Create reusable template:**
1. Build perfectly formatted document once (following specs above)
2. Save as `.dotx` (Word template)
3. Use template for new projects
4. Replace placeholder text

**Advantages:**
- Faster than manual each time
- Consistent formatting
- Shareable with team

**Limitations:**
- Still manual content entry
- No automatic validation
- Character limits not enforced

## Technical Notes for Programmatic Generation

**Key functions needed:**
- `Document()` - Create document
- `add_paragraph()` - Add text elements
- `add_table()` - Create tables
- `add_page_break()` - Force page break

- `set_cell_margins()` - Custom function for padding

**DXA to Points Conversion:**
- 1 point = 20 DXA
- 1 inch = 1440 DXA = 72 points

**EMU (English Metric Units):**
- Used by python-docx for some measurements
- 914400 EMU = 1 inch
- 635 EMU = 1 DXA

## Questions and Support

### Common Questions

**Q: Can I use a different font?**
A: No. Calibri is specified for BCG-level consistency across Vianeo framework.

**Q: Can I add more than 10 players/influencers?**
A: No. Strategic selection of 8-10 most critical is a core principle of Version 2.0.

**Q: What if my strategic notes are longer than 250 characters?**
A: Edit for conciseness. Use semicolons, remove articles, lead with most important info.

**Q: Can the document be longer than 2 pages?**
A: No. The 2-page constraint forces strategic focus and executive-friendly brevity.

**Q: Should I include both tables on one page if they fit?**
A: No. Page break is mandatory to maintain clear section separation.
