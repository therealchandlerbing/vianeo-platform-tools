## Dimension 1: LEGITIMACY

**Weight: 15%** | **Threshold: 3.0** | **Questions: 2**

**Formula**: (Q8 + Q13) / 2

### Q8: Problem Definition

**Question**: How well is the core problem defined and validated?

**Minimum for Score 3**: Problem validated with 5-10 stakeholders, basic evidence of significance

| Score | Criteria | Evidence Required |
|-------|----------|------------------|
| 5 | 20+ stakeholders validate problem | Interview notes, problem significance data, multiple validation sources |
| 4 | 10-15 stakeholders validate problem | Interview documentation, impact quantified |
| 3 | 5-10 stakeholders validate problem | Interview notes, initial impact assessment |
| 2 | 1-3 stakeholders, mostly internal | Limited documentation |
| 1 | No external validation | Need: 5-10 stakeholder interviews validating problem |

**Red Flag**: Score 1 combined with Q13 = 1 suggests problem may not exist

### Q13: Problem Validation

| Score | Criteria | Evidence Required |

| 4 | 10-15 stakeholders validate | Consistent feedback about significance |
| 3 | 5-10 stakeholders validate | Emerging patterns of significance |
| 2 | 1-4 stakeholders mention | Inconsistent feedback |

## Critical Patterns to Watch

### Pattern 1: No Customer Validation
- Q7 = 1 (no interviews)
- Q13 = 1 (problem not validated)
- Q22 = 1 (no testing per segment)

**Action**: STOP. Conduct customer discovery immediately.

### Pattern 2: Ecosystem Blindness
- Q3 = 1 (no ecosystem map)
- Q17 = 1 (no barriers identified)
- Q20 = 1 (no market understanding)

**Action**: Map ecosystem before proceeding.

### Pattern 3: Untested Business Model
- Q14 = 1 (no model defined)
- Q19 = 1 (not tested)
- Q27 = 1 (no value capture)

**Action**: Define and test business model immediately.

### Pattern 4: Resource Gaps
- Q1 = 1 (lacking resources)
- Q16 = 1 (no team)
- Q26 = 1 (no plan)

**Action**: Secure resources before proceeding.

## Scoring Quick Reference

**For "How Many" Questions** (Q7, Q22):
- Score 5: 20+ per segment
- Score 4: 15-20 per segment
- Score 3: 5-10 per segment ← Minimum threshold
- Score 2: 1-4 per segment
- Score 1: 0

**For "How Well" Questions** (Q2, Q5, Q9, Q13, Q25):
- Score 5: Extensively validated with 20+ external sources
- Score 4: Well validated with 10-15 external sources
- Score 3: Basically validated with 5-10 external sources ← Minimum threshold
- Score 2: Poorly validated with 1-4 external sources
- Score 1: Not validated

**For "Is There" Questions** (Q1, Q14, Q15, Q16, Q26, Q27, Q29):
- Score 5: Complete, detailed, validated
- Score 4: Clear, developed, mostly validated
- Score 3: Basic, defined, somewhat validated ← Minimum threshold
- Score 2: Partial, unclear, not validated
- Score 1: Absent, not defined
